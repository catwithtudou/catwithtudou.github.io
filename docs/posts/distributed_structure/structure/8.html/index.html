APM组件选型

  1. 探针的性能

    重点在agent对服务的吞吐量、CPU和内存的影响。

    微服务的规模和动态性使数据收集的成本提高。

  2. collector的可扩展性

    水平扩展以便支持更大规模服务器集群。

  3. 全面的调用链路数据分析

    提供代码级别的可见性，定位失败点和瓶颈。

  4. 对于开发透明，容易开关

    无需修改代码添加新功能，容易启用或者禁用。

  5. 完整的调用链应用拓扑

    自动检测应用拓扑，应用架构。
